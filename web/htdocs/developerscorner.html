JDEE Developer's Corner

Welcome to the JDEE Developer's Corner. This page explains what you need to do to participate in the development of the JDEE.

Becoming a JDEE Developer

To become a JDEE developer:

  1. Subscribe to the JDE developer mailing lists.

  2. Install cvs client software on your development system (see Using the JDE Source Code Repository).

  3. Create a JDEE sandbox on your development system.

  4. Download and install Bugtrack.

    Bugtrack is the Emacs-based bugtracking system used to track JDEE bugs and new feature and feature enhancement requests. The Bugtrack database for the JDEE, BUGS, is in the JDEE's root directory. Use Bugtrack as your to-do list.

  5. Discuss potential assignments with the JDEE development team leader.

  6. Implement and test features/bug fixes in your sandbox.

  7. Update the source repository.

  8. Post a notice to jde-devel describing the changes you have made.

Using the JDEE Source Code Repository

About the Source Code Repository

The JDEE source code repository contains the latest development version of the JDEE source code. The repository is maintained by Sunsite Denmark and controlled by the Concurrent Version System (CVS) system. Anyone with Internet access and CVS client software can check source files out of the repository. Only authorized persons can update the repository.

Installing CVS Client Software

You need to install CVS client software on your system. The client allows you to check source files out of the repository, and, if you have commit access, commit updated files to the repository. CVS client software is widely and freely available. If you plan to develop under Windows, I recommend that you use Cyclic's Windows port of the CVS client. The CVS client is a command-line application. The pcl-cvs package provides an Emacs interface to the client.

Updating the Repository

Developers can update the source code in the repository by

  • posting patches to the JDEE patches mailing list
  • committing updated files to the repository via CVS

Only developers with commit access to the repository can submit changes directly. Everyone else must submit changes to the patches mailing list. The changes are reviewed and committed by developers with commit accces.

Requesting Commit Access to the Repository

You may request commit access to the repository. Before doing so, it is a good idea to first submit a few changes via the patches mailing list to establish your credentials as a developer. Then, email your request to the repository administrator. Your request should specifiy the following information:

  • user name by which you would like to be known to the repository
  • password to gain access to the repository (specify if the password is crypt-encoded)
  • email address to which repository notifications should be sent

You will be notified by email when you have been granted commit access.

Logging onto the Repository

If you have commit access, you can log into the repository as yourself. To log in as yourself, open a command shell and enter:

cvs -d :pserver:USERNAME@sunsite.dk:/pack/anoncvs login

where USERNAME is your user name. The cvs client prompts you to enter your password.

Anyone can login into the repository anonymously. As an anonymous user, you have read but not write access to the repository. To log in anonymously, enter

cvs -d :pserver:cvs@sunsite.dk:/pack/anoncvs login

Then, enter cvs at the password prompt.

Creating a JDEE Sandbox

To create a sandbox, log into the repository. Next change to the directory on your system in which you want to create the sandbox. Then check out the JDEE source code. The following example illustrates how to create a sandbox, using a command shell:

> cvs -d :pserver:cvs@sunsite.dk:/pack/anoncvs login
(Logging in to cvs@sunsite.dk)
CVS password: cvs
> mkdir jde-sandbox
> cd jde-sandbox
> cvs -d :pserver:cvs@sunsite.dk:/pack/anoncvs co jde
... (checkout messages)
> cvs -d :pserver:cvs@sunsite.dk:/pack/anoncvs logout

Communicating with Other JDEE Developers

JDE developers use the following mailing lists to coordinate development activities:

  • jde-devel

    Used by developers to discuss JDE development issues and business. The JDE source code repository sends commit and watch notifications to this mailing list as well. To subscribe, send an empty message to jde-devel-subscribe. To unsubscribe, send an empty message to jde-devel-unsubscribe.

  • jde-patches

    Used by developers who do not commit access to the repository to post patches to the JDE source code. Patches posted to this mailing list are reviewed and committed by members who do have commit access. To subscribe, send an empty message to jde-patches-subscribe. To unsubscribe, send an empty message to jde-patches-unsubscribe.
